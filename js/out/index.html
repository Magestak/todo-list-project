Documentation technique - Application "todos"

Ce document (à destination des développeurs), à pour vocation de servir de support technique.
Il décrit et détaille l'architecture de l'application, ainsi que les différentes classes, méthodes et fonctions globales qui servent à son fonctionnement.

Architecture de l'application

Organisation dossiers et fichiers de l'application "todos"

Ce schéma matérialise la structure de l'application "todos" qui comprend (entre autres) les dossiers et fichiers principaux suivants:

  • index.html: ce fichier matérialise l'application dans le navigateur.
  • package.json: ce fichier liste les dépendances à installer avec npm nécessaires à l'application (jasmine, jsdoc).
  • js: ce dossier comprend les fichiers javascript de l'application.
    • app.js: Ce fichier est le point d’entrée de l’application. Il permet d’initialiser la base de données (store), le modèle (model), le template (template), la vue (view) et le contrôleur (controller).
    • store.js: Ce fichier permet la création de la base de données de l’application. Les données saisis par l'utilisateur seront stockées dans le "localStorage" du navigateur. Les méthodes de ce fichier ne sont appelées que dans le fichier model.js.
    • model.js: Ce fichier sert à gérer les données. Il s’occupe de récupérer et de gérer les informations dans la base de données. Il créé un nouveau modèle de base de données en se connectant au LocalStorage du navigateur (via store). Il utilisera les méthodes du fichier Store.js pour effectuer les opérations sur la base de données, qui seront appelées via le contrôleur.
    • view.js: Ce fichier gère l’affichage et la manipulation du DOM. Les changements d’états du template HTML s’effectuent ici. Ce fichier permet de mettre à jour le visuel de l’application. La fonction view permet deux types d’actions : attacher un évènement d'une tâche donnée avec un gestionnaire d'évènement (bind) et faire le rendu d'une commande avec ses options (render).
    • controller.js: Ce fichier permet de faire le lien entre la base de données (Model) et l’interface du site (view) selon les actions de l’utilisateur
    • helpers.js: Ce fichier permet de simplifier la syntaxe de JavaScript, Il crée des fonctions réutilisables et fournit ainsi des raccourcis utilisés dans les principaux fichiers.
    • template.js: Ce fichier permet de fournir une trame HTML pour chaque nouvelle entrée ajoutée. Le template est gérée et actualisée par le fichier view.js.
  • nodes_modules: ce dossier comprend les packages node (dépendances) nécessaires au fonctionnement de l'application.
  • test: ce dossier reprend les tests réalisés pour la construction de l'application.

Architecture MVC

L’architecture de l’application est basée sur une architecture MVC (Modèle-vue-contrôleur).

MVC est composé de trois composants

  • Le Modèle est l'endroit où les objets de données de l'application sont stockés. Le modèle ne sait rien des vues et des contrôleurs. Lorsqu'un modèle change, il informe généralement ses observateurs qu'un changement s'est produit.
  • La Vue est ce qui est présenté aux utilisateurs et comment les utilisateurs interagissent avec l'application.
  • Le Controleur est le décideur et le lien entre le modèle et la vue. Le contrôleur met à jour la vue lorsque le modèle change. Il ajoute également des écouteurs d'événements à la vue et met à jour le modèle lorsque l'utilisateur manipule la vue.
